# Neovim Editing 練習帳

[editing-cheatsheet](/Users/sfukunaga/ghq/github.com/hforever11/dotfiles/docs/nvim/editing-cheatsheet.md) の内容を手で動かして覚えるための課題集。

## 使い方

1. このファイルを Neovim で開く
2. 各課題の **対象** ブロックにカーソルを置く
3. **目標** の形になるよう、自力で編集する
4. 詰まったら **解答キー** を見る（`<Esc>` などの特殊キーはそのまま打鍵）

> 壊しても困らないよう、編集はこのファイルのコピーで行うのがおすすめ。
> `:w !cat` で保存せず確認したり、`u` でいつでも戻せる。

この設定固有の注意（チートシート参照）:

- `x` は **レジスタを汚さない削除オペレータ**。1 文字消すだけなら使えない（`xx` で行、`xiw` で単語）
- マクロは `qq` 記録 → `q` 停止 → **`Q` で再生**（`@q` 相当）
- `/` は **very magic**（`\v`）。`(` `+` `|` はメタ文字
- 数値の増減は `<leader>+` / `<leader>-`（`<C-a>` / `<C-x>` のラッパ）

## 1. テキストオブジェクト

### 1-1. 単語を入れ替える

**目標**: `oldName` を `newName` に。

**対象**:

```text
const result = oldName + 1
```

**解答キー**: `oldName` の上で `ciwnewName<Esc>`

### 1-2. クォートの中身だけ変える

**目標**: `"foo"` を `"bar"` に。クォートは残す。

**対象**:

```text
const label = "foo"
```

**解答キー**: 行内で `ci"bar<Esc>`（`ci"` はどこからでも効く）

### 1-3. 関数の中身を消す（Treesitter）

**目標**: `greet` 関数の本体だけ空にする。

**対象**:

```text
function greet() {
  console.log("hi")
  console.log("bye")
}
```

**解答キー**: 関数内で `cif`（中身を変更）/ 消すだけなら `dif`

## 2. Surround

### 2-1. 単語をクォートで囲む

**目標**: `hello` → `"hello"`。

**対象**:

```text
hello
```

**解答キー**: `ysiw"`

### 2-2. クォートをシングルに変える

**目標**: `"hello"` → `'hello'`。

**対象**:

```text
"hello"
```

**解答キー**: `cs"'`

### 2-3. タグを外す

**目標**: `<div>hello</div>` → `hello`。

**対象**:

```text
<div>hello</div>
```

**解答キー**: `dst`

## 3. Dot Repeat（`.`）

### 3-1. 行末にセミコロンを足す

**目標**: 3 行すべての行末に `;`。

**対象**:

```text
const a = 1
const b = 2
const c = 3
```

**解答キー**: 1 行目で `A;<Esc>`、続けて `j.` を 2 回

### 3-2. cgn で同じ単語を順に書き換える

**目標**: すべての `get` を `fetch` に。

**対象**:

```text
getUser()
getOrder()
getProfile()
```

**解答キー**: `/get<CR>` → `cgnfetch<Esc>` → `.` `.`
（カーソルが `get` に乗っていれば `*` で検索を始めるのが速い）

### 3-3. 箇条書きをチェックボックスにする

**目標**: 各行頭の `-` を `- [ ]` に。

**対象**:

```text
- task1
- task2
- task3
```

**解答キー**: 1 行目で `^a [ ]<Esc>`、続けて `j.` を 2 回

## 4. マクロ（基本）

### 4-1. リストを配列要素にする（複数ステップ）

**目標**: 各行を `"value",` の形に。`.` 1 回では届かない（接頭辞＋接尾辞の 2 編集）。

**対象**:

```text
apple
banana
cherry
```

**解答キー**:

```vim
qq             " 記録開始
I"<Esc>        " 行頭に "
A",<Esc>       " 行末に ",
j              " 次の行へ
q              " 停止
2Q             " 残り 2 行へ
```

**期待結果**:

```text
"apple",
"banana",
"cherry",
```

### 4-2. 検索を混ぜて飛び石を処理する

**目標**: `log` で始まる行だけ `// ` でコメントアウト。`ok()` 行は触らない。

**対象**:

```text
log(a)
ok()
log(b)
ok()
log(c)
```

**解答キー**:

```vim
qq             " 記録開始
/log<CR>       " 次の log へ
I// <Esc>      " 行頭にコメント
q              " 停止
99Q            " 一致が尽きるまで（検索失敗で停止）
```

### 4-3. 行を複製して連番を振る（マクロより速い手）

**目標**: 1 行を 5 行に増やし、番号を 1→5 に。

連番付けは **マクロ不要**。`g<C-a>`（dial のビジュアル連番）が最短。
`<C-a>` が「全部 +1」なのに対し、`g<C-a>` は選択行を 1,2,3… と**階段状**に増やす。

**対象**:

```text
test_case_0
```

**解答キー**:

```vim
yy4p           " 行を 5 行に複製
ggVG           " 全行をビジュアル選択
g<C-a>         " 0,0,0,0,0 → 1,2,3,4,5 の連番に
```

> どうしてもマクロでやるなら `qqyyp<C-a>q` を `4Q`。ただし上の `g<C-a>` の方が速い。

### 4-4. dial でトグル・日付・semver

**目標**: カーソル下のフィールドを `<C-a>` / `<C-x>` で増減・トグルする（数値以外も対象）。

**対象**:

```text
enabled = false
version = "v1.2.9"
release = 2024-01-31
mode    = a && b
```

**解答キー**: 対象語にカーソルを置いて `<C-a>`

- `true` → `false`（`<C-a>` でトグル）
- `v1.2.9` の `9` の上 → `v1.2.10`（patch をバンプ）
- `2024-01-31` は **カーソルを置いた桁**で増減（年/月/日）
- `a && b` の `&&` の上 → `||`

数値の連番は 4-3 の `g<C-a>`、単発の増減・トグルはこの `<C-a>` を使い分ける。

## 5. マクロ（上級）

### 5-1. 再帰マクロ（回数を数えない）

**目標**: 行数を数えずに 4-1 と同じ変換を最後まで自走させる。

**対象**:

```text
red
green
blue
yellow
```

**解答キー**:

```vim
qqq            " q レジスタを空にする
qq             " 記録開始
I"<Esc>A",<Esc>" 1 行分の処理
j              " 次の行へ
@q             " 自分を呼ぶ（記録中は空振りでよい）
q              " 停止
Q              " 一度実行 → 最後まで止まらない
```

### 5-2. マッチ行だけに一括適用（`:g` + マクロ）

**目標**: `TODO` を含む行だけ末尾に ` <-- check` を付ける。

**対象**:

```text
clean this up TODO
this line is fine
fix later TODO
done
```

**解答キー**:

```vim
qqq                          " 空にする
qqA <-- check<Esc>q          " 「行末に追記」だけのマクロを記録
:g/TODO/normal @q            " TODO 行だけに適用
```

### 5-3. 範囲指定で適用（`:normal`）

**目標**: ビジュアルで選んだ行すべてを `// ` でコメントアウト。

**対象**:

```text
line one
line two
line three
```

**解答キー**:

```vim
qqI// <Esc>q     " 行頭にコメントを入れるだけのマクロ
" 対象を V で複数行選択してから:
:'<,'>normal @q
```

## 6. 仕上げ（組み合わせ）

### 6-1. 検索 → 変更 → 戻る

**目標**: `targetFn` の呼び出しまで飛び、引数 `1` を `2` にして、元の位置へ戻る。

**対象**:

```text
start here
// ... 省略 ...
targetFn(1)
```

**解答キー**: `/targetFn<CR>` → `f1` → `<leader>+`（または `ci(2<Esc>`）→ `<C-o>` で戻る

### 6-2. ヤンク履歴ピッカー

**目標**: 何回かヤンク・削除した後、少し前にヤンクした内容を貼り直す。

**手順**: 適当に `yiw` を数回 → `<leader>y`（Snacks UI で表示されるヤンク履歴）→ 目的の項目を選んで貼り付け
