# Neovim Editing Cheatsheet

編集・移動・テキストオブジェクト系のメモをまとめたもの。

## まず覚える移動

普段のコーディングで使用頻度が高いものから覚える。

| 用途           | キー                 | 使いどころ                     |
| -------------- | -------------------- | ------------------------------ |
| 画面を大きく動く | `<C-u>` / `<C-d>`   | 読み進める、レビューする       |
| 単語単位で動く | `w` / `b` / `e`      | 近距離の移動                   |
| 行内で狙う     | `f{char}` / `t{char}` | 同じ行の目的位置へ素早く飛ぶ   |
| 検索で飛ぶ     | `/`, `n`, `N`        | 中距離以上の移動               |
| 定義へ飛ぶ     | `gd`, `gr`           | 実装や参照を追う               |
| 戻る           | `<C-o>`              | ジャンプ後に元の場所へ戻る     |

`hjkl` は微調整用で、主役は検索・単語移動・ジャンプ。

## 構造単位の移動

コードでは「関数」「クラス」「引数」など、意味のある単位で飛ぶことが多い。

| 単位   | キー                 | 用途                   |
| ------ | -------------------- | ---------------------- |
| 関数   | `]f`, `[f`, `]F`, `[F` | 関数の先頭 / 末尾へ移動 |
| クラス | `]c`, `[c`, `]C`, `[C` | クラスの先頭 / 末尾へ移動 |
| 引数   | `]a`, `[a`           | 長い関数呼び出しの引数移動 |

この repo でも Treesitter textobjects で有効化している。

補足:

- `]f` / `[f` は便利だが、押しやすい配置ではない
- 頻度が高ければ後で別キーへリマップしてよい
- ただし最初は `/` 検索と `gd` の方が投資対効果は高い

## Navigation

`hjkl` 連打ではなく、画面移動・検索・ジャンプを主に使う。

### 画面単位

| キー    | 動作                     |
| ------- | ------------------------ |
| `<C-u>` | 半ページ上               |
| `<C-d>` | 半ページ下               |
| `<C-f>` | 1ページ下                |
| `<C-b>` | 1ページ上                |
| `zz`    | カーソル位置を画面中央に |
| `zt`    | カーソル位置を画面上部に |
| `zb`    | カーソル位置を画面下部に |

### 行内移動

| キー      | 動作                           |
| --------- | ------------------------------ |
| `f{char}` | 行内で文字にジャンプ           |
| `F{char}` | 行内で逆方向に文字にジャンプ   |
| `t{char}` | 文字の手前にジャンプ           |
| `T{char}` | 逆方向で文字の手前にジャンプ   |
| `;`       | 直前の `f/t` を繰り返す        |
| `,`       | 直前の `f/t` を逆方向に繰り返す |
| `w` / `W` | 次の単語先頭へ                 |
| `b` / `B` | 前の単語先頭へ                 |
| `e` / `E` | 単語末尾へ                     |
| `0`       | 行頭へ                         |
| `^`       | 行頭（非空白）へ               |
| `$`       | 行末へ                         |

### ファイル内ジャンプ

| キー         | 動作                                      |
| ------------ | ----------------------------------------- |
| `/{pattern}` | 前方検索                                  |
| `?{pattern}` | 後方検索                                  |
| `n` / `N`    | 次/前の検索結果へ                         |
| `{数字}j`    | 相対行番号を見て下にジャンプ（例: `12j`） |
| `{数字}k`    | 相対行番号を見て上にジャンプ              |
| `{`          | 前の段落へ                                |
| `}`          | 次の段落へ                                |
| `gg`         | ファイル先頭へ                            |
| `G`          | ファイル末尾へ                            |
| `{数字}G`    | 指定行へ（例: `42G`）                     |
| `gd`         | 定義へジャンプ（LSP）                     |
| `%`          | 対応する括弧へ                            |

### 履歴ベース

| キー    | 動作                         |
| ------- | ---------------------------- |
| `<C-o>` | ジャンプリストで前に戻る     |
| `<C-i>` | ジャンプリストで次に進む     |
| `gi`    | 最後に Insert した位置に戻る |
| `g;`    | 変更リストで前の変更位置へ   |
| `g,`    | 変更リストで次の変更位置へ   |

### マーク

| キー         | 動作                           |
| ------------ | ------------------------------ |
| `m{a-z}`     | ローカルマークを設定           |
| `'{a-z}`     | マーク位置の行頭へジャンプ     |
| `` `{a-z} `` | マーク位置の正確な位置へジャンプ |
| `''`         | 直前のジャンプ位置に戻る       |

## Surround

### 基本操作

| 操作 | コマンド           | 説明                     |
| ---- | ------------------ | ------------------------ |
| 追加 | `ys{motion}{char}` | motion 範囲を char で囲む |
| 削除 | `ds{char}`         | char の囲みを削除        |
| 変更 | `cs{old}{new}`     | 囲みを old から new に変更 |

### 追加

| コマンド        | 前            | 後                          |
| --------------- | ------------- | --------------------------- |
| `ysiw"`         | `hello`       | `"hello"`                   |
| `ysiw'`         | `hello`       | `'hello'`                   |
| `ysiw)`         | `hello`       | `(hello)`                   |
| `ysiw(`         | `hello`       | `( hello )`                 |
| `ysiw]`         | `hello`       | `[hello]`                   |
| `ysiw[`         | `hello`       | `[ hello ]`                 |
| `ysiw}`         | `hello`       | `{hello}`                   |
| `ysiw{`         | `hello`       | `{ hello }`                 |
| `ysiw>`         | `hello`       | `<hello>`                   |
| `ysiw<div>`     | `hello`       | `<div>hello</div>`          |
| `ysiwt` + `div` | `hello`       | `<div>hello</div>`          |
| `ysiw`` `       | `hello`       | `` `hello` ``               |
| `yss"`          | `hello world` | `"hello world"`             |

### 削除 / 変更

| コマンド   | 前                 | 後                 |
| ---------- | ------------------ | ------------------ |
| `ds"`      | `"hello"`          | `hello`            |
| `ds)`      | `(hello)`          | `hello`            |
| `dst`      | `<div>hello</div>` | `hello`            |
| `cs"'`     | `"hello"`          | `'hello'`          |
| `cs")`     | `"hello"`          | `(hello)`          |
| `cst"`     | `<div>hello</div>` | `"hello"`          |
| `cs"t` + `span` | `"hello"`      | `<span>hello</span>` |

### ビジュアルモード

| コマンド   | 説明                           |
| ---------- | ------------------------------ |
| `S{char}`  | 選択範囲を char で囲む         |
| `gS{char}` | 選択範囲を char で囲む（改行付き） |

### motion / alias

| 項目  | 対象                    |
| ----- | ----------------------- |

### 移動

| コマンド | 説明               |
| -------- | ------------------ |
| `]f`     | 次の関数の先頭へ   |
| `[f`     | 前の関数の先頭へ   |
| `]F`     | 次の関数の末尾へ   |
| `[F`     | 前の関数の末尾へ   |
| `]c`     | 次のクラスの先頭へ |

### 覚え方

- `a` = around
- `i` = inside
- `f` = function
- `c` = class
- `]` = 次へ
- `[` = 前へ

## Dot Repeat

`.` は「直前の変更を繰り返す」コマンド。移動と変更を分けると強い。

基本パターン:

1. 1回だけ丁寧に変更する
2. 検索や移動で次の場所へ飛ぶ
3. `.` で同じ変更を繰り返す

### まず覚える使い方

| パターン | 例 | 何が起きるか |
| --- | --- | --- |
| 単語を書き換える | `ciwfoo<Esc>` | カーソル下の単語を `foo` に変更 |
| 行末に追加する | `A;<Esc>` | 行末に `;` を追加 |
| 単語を削除する | `dw` | 次の単語まで削除 |
| 囲みを変更する | `cs\"'` | `"` を `'` に変更 |

繰り返し例:

```vim
ciwfoo<Esc>
w
.
w
.
```

### 検索と組み合わせる

`n` と `.` の組み合わせが実戦的。

```vim
/pattern
ciwreplacement<Esc>
n
.
n
.
```

「同じ種類の修正を次の一致箇所にも適用する」ときに使う。

### cgn が強い

検索一致そのものを書き換えるなら `cgn` が強い。

```vim
/old
cgnnew<Esc>
.
.
.
```

最初の `cgn` で「次の検索一致を変更」して、以後 `.` で次の一致へ同じ変更を繰り返せる。

### TypeScript でよくある例

```ts
const foo = 1
const bar = 2
const baz = 3
```

各行末に `;` を付けたい:

```vim
A;<Esc>
j
.
j
.
```

関数名を段階的に変えたい:

```ts
getUser()
getOrder()
getProfile()
```

```vim
/get
cgnfetch<Esc>
.
.
```

### Lua でよくある例

```lua
vim.keymap.set("n", "x", "dd")
vim.keymap.set("n", "y", "yy")
vim.keymap.set("n", "p", "pp")
```

引数の文字列だけ順に直したいとき:

```vim
f"
ci"delete<Esc>
n
.
```

または同種の式なら `ci"` と `.` を繰り返す。

### Markdown でよくある例

箇条書きをチェックボックスに変える:

```md
- task1
- task2
- task3
```

```vim
^r[ ]<Esc>
j
.
j
.
```

単語を backtick で囲む:

```vim
ysiw`
w
.
```

### 向いているケース

- 同じ変更を複数箇所へ適用したい
- 検索で次の対象へ飛べる
- text object で対象を安定して取れる

### 向いていないケース

- 毎回入力内容が大きく違う
- 複数ステップを丸ごと再現したい
- 位置取りまで含めて自動化したい

そういうときはマクロの方が向いている。

## Search

この設定では検索キーマップを少し変えている。

| キー | 実際の挙動 |
| --- | --- |
| `/` | `very magic` 検索 |
| `?` | リテラル検索 (`very nomagic`) |
| `<leader>nh` | 検索ハイライトをクリア |

つまり:

- `/` は正規表現を短く書きやすい
- `?` は記号をそのまま探したいときに向く

### 検索で意識すること

- 検索は「探す」より「移動する」ために使う
- 完全一致を狙いすぎず、近くまで飛べれば十分
- 検索後は `n`, `N`, `f`, `w`, `b` で微調整する
- 毎回打ち直すより、1回検索して `n` / `N` を使い回す
- ジャンプ後に戻るために `<C-o>` をセットで使う

### 使い分け

#### `/` を使うとき

- 単語の一部で飛びたい
- 正規表現を使いたい
- 複数候補を順に見たい

例:

```vim
/func
/const\s+
/TODO|FIXME
```

`very magic` なので、`+`, `|`, `()` などを比較的そのまま書ける。

#### `?` を使うとき

- 上方向へ探したい
- 記号をそのまま探したい
- 正規表現として解釈されたくない

例:

```vim
?foo.bar
?items[0]
?user?.name
```

`?` は `very nomagic` にしているので、`.` や `[` をそのまま扱いやすい。

### 実践パターン

#### 移動

```vim
/handle
n
n
N
```

1回検索して、次候補は `n` / `N` で辿る。

#### 編集と組み合わせる

```vim
/old
cgnnew<Esc>
.
.
```

検索一致を順に同じやり方で書き換える。

#### ジャンプして戻る

```vim
/someFunction
gd
<C-o>
```

検索や定義ジャンプの後は `<C-o>` で戻る。

### 検索語の作り方

- 長い文字列を入れすぎない
- 2〜6文字くらいで候補が絞れれば十分
- 一意でなくても `n` で進めばいい
- 近距離なら `f{char}` や `t{char}` の方が速い

### 向いている流れ

1. `/` または `?` で近くへ飛ぶ
2. `n` / `N` で目的位置まで寄せる
3. `ciw`, `cgn`, `.` などの変更へつなげる
