# Neovim Editing Cheatsheet

編集・移動・テキストオブジェクト系のメモをまとめたもの。

## まず覚える移動

普段のコーディングで使用頻度が高いものから覚える。

| 用途           | キー                 | 使いどころ                     |
| -------------- | -------------------- | ------------------------------ |
| 画面を大きく動く | `<C-u>` / `<C-d>`   | 読み進める、レビューする       |
| 単語単位で動く | `w` / `b` / `e`      | 近距離の移動                   |
| 行内で狙う     | `f{char}` / `t{char}` | 同じ行の目的位置へ素早く飛ぶ   |
| 検索で飛ぶ     | `/`, `n`, `N`        | 中距離以上の移動               |
| 単語で検索する | `*` / `#`            | カーソル下の単語で前方/後方検索 |
| 定義へ飛ぶ     | `gd`, `gr`, `gI`, `gy` | 定義・参照・実装・型定義を追う |
| 戻る           | `<C-o>`              | ジャンプ後に元の場所へ戻る     |

`hjkl` は微調整用で、主役は検索・単語移動・ジャンプ。

## 構造単位の移動

コードでは「関数」「クラス」「引数」など、意味のある単位で飛ぶことが多い。

| 単位     | キー                 | 用途                   |
| -------- | -------------------- | ---------------------- |
| 関数     | `]f`, `[f`, `]F`, `[F` | 関数の先頭 / 末尾へ移動 |
| クラス   | `]c`, `[c`, `]C`, `[C` | クラスの先頭 / 末尾へ移動 |
| 引数     | `]a`, `[a`           | 長い関数呼び出しの引数移動 |
| git hunk | `]h`, `[h`, `]H`, `[H` | 変更箇所単位の移動（gitsigns） |

この repo でも Treesitter textobjects で有効化している。
hunk はテキストオブジェクト `ih` も使える（例: `<leader>ghs` の代わりに `vih` で選択して確認）。

補足:

- `]f` / `[f` は便利だが、押しやすい配置ではない
- 頻度が高ければ後で別キーへリマップしてよい
- ただし最初は `/` 検索と `gd` の方が投資対効果は高い

## Navigation

`hjkl` 連打ではなく、画面移動・検索・ジャンプを主に使う。

| ------- | ------------------------ |

| `<C-d>` | 半ページ下（カーソルを画面中央に保つ） |

| --------- | ------------------------------ |
| `f{char}` | 行内で文字にジャンプ           |
| `F{char}` | 行内で逆方向に文字にジャンプ   |

## React / TypeScript / Python の便利コマンド

前提プラグイン: Treesitter textobjects / nvim-surround / Comment.nvim +
ts-context-commentstring / nvim-ts-autotag。下表はすべて実機で動作確認済み。

### 共通（関数・クラス・引数）

オペレータ（`d` 削除 / `c` 変更 / `y` ヤンク / `v` 選択）と組み合わせる。

| text object | 対象 | 例（`d` と組む） |
| --- | --- | --- |
| `if` / `af` | 関数の中身 / 関数全体 | `dif` で本体だけ空に、`daf` で関数ごと削除 |
| `ic` / `ac` | クラスの中身 / クラス全体 | `cic` で中身を書き換え、`dac` でクラス削除 |
| `ia` / `aa` | 引数1つ / 引数（カンマ込み） | `dia` で値だけ、`daa` で区切りごと削除 |

移動（カーソルだけ動かす）:

| キー | 動作 |
| --- | --- |
| `]f` / `[f` | 次 / 前の関数の先頭へ |
| `]c` / `[c` | 次 / 前のクラスの先頭へ |
| `]a` / `[a` | 次 / 前の引数へ |

### React（JSX / TSX）

| コマンド | 動作 | 例 |
| --- | --- | --- |
| `cit` / `dit` | タグの**中身**を変更 / 削除 | `<span>X</span>` の `X` |
| `cat` / `dat` | **要素全体**を変更 / 削除 | `<span>…</span>` ごと |
| `dst` | 囲みタグを剥がす | `<b>x</b>` → `x` |
| `cst{tag}<CR>` | 囲みタグを変更 | `cstb<CR>`: `<span>` → `<b>` |
| `ys{motion}t{tag}<CR>` | 要素をタグで囲む | `ysiwtem<CR>`: `hello` → `<em>hello</em>` |
| `ci"` / `ci'` | 属性値を変更 | `className="x"` の `x` |
| `gcc` / `gc{motion}` | コメント化（**文脈判定**） | JSX 内は `{/* */}`、JS 部は `//` |

`gc` は ts-context-commentstring により、同じファイル内でも JSX の子要素なら
`{/* … */}`、ロジック部なら `//` に自動で切り替わる。

autotag の挙動（コマンドではなく自動）:

- 開始タグ名を編集すると**閉じタグも自動で追従**する
- `<div>` と打つと `</div>` が自動補完される

## Increment / Decrement（dial.nvim）

`<C-a>` / `<C-x>` を dial.nvim で拡張している。数値だけでなく、カーソル下の
「増やせるもの」を文脈で判定して増減・トグルする。

| キー | モード | 動作 |
| --- | --- | --- |
| `<C-a>` / `<C-x>` | normal | カーソル下を増減 / トグル |
| `<leader>+` / `<leader>-` | normal | 同上（別名） |
| `<C-a>` / `<C-x>` | visual | 選択内の数値をまとめて増減 |
| `g<C-a>` / `g<C-x>` | visual | 選択行を 1,2,3… の **連番**に（階段状） |

対象（カーソル下のフィールドを判定）:

| 対象 | 例 | `<C-a>` 後 |
| --- | --- | --- |
| 10進 | `42` | `43` |
| 16進 | `0xff` | `0x100` |
| semver | `v1.2.9`（`9` 上） | `v1.2.10` |
| 日付 | `2024-01-31` | カーソルの桁で年/月/日を増減 |
| 真偽値 | `true` / `True` | `false` / `False` |
| 論理 | `&&` / `==` / `and` | `\|\|` / `!=` / `or` |

### 連番の作り方

同じ数字を並べて `g<C-a>` が定石。マクロより速い。

```vim
yy4p     " 行を 5 行に複製
ggVG     " 全行を選択
g<C-a>   " 0,0,0,0,0 → 1,2,3,4,5
```

### 使い分け

- 単発の増減・トグル → `<C-a>` / `<C-x>`（対象語にカーソルを乗せる）
- 連番を振る → visual + `g<C-a>`
- 全部に同じ数を足す → visual + `<C-a>`

## Dot Repeat

`.` は「直前の変更を繰り返す」コマンド。移動と変更を分けると強い。

基本パターン:

1. 1回だけ丁寧に変更する
2. 検索や移動で次の場所へ飛ぶ
3. `.` で同じ変更を繰り返す

### まず覚える使い方

| パターン | 例 | 何が起きるか |
| --- | --- | --- |
| 単語を書き換える | `ciwfoo<Esc>` | カーソル下の単語を `foo` に変更 |
| 行末に追加する | `A;<Esc>` | 行末に `;` を追加 |
| 単語を削除する | `dw` | 次の単語まで削除 |
| 囲みを変更する | `cs\"'` | `"` を `'` に変更 |

繰り返し例:

```vim
ciwfoo<Esc>
w
.
w
.
```

### 検索と組み合わせる

`n` と `.` の組み合わせが実戦的。

```vim
/pattern
ciwreplacement<Esc>
n
.
n
.
```

「同じ種類の修正を次の一致箇所にも適用する」ときに使う。

### cgn が強い

検索一致そのものを書き換えるなら `cgn` が強い。

```vim
/old
cgnnew<Esc>
.
.
.
```

最初の `cgn` で「次の検索一致を変更」して、以後 `.` で次の一致へ同じ変更を繰り返せる。

対象の単語にカーソルが乗っているなら `*` で検索を始めるのが最速:

```vim
*
cgnnew<Esc>
.
.
```

### TypeScript でよくある例

```ts
const foo = 1
const bar = 2
const baz = 3
```

各行末に `;` を付けたい:

```vim
A;<Esc>
j
.
j
.
```

関数名を段階的に変えたい:

```ts
getUser()
getOrder()
getProfile()
```

```vim
/get
cgnfetch<Esc>
.
.
```

### Lua でよくある例

```lua
vim.keymap.set("n", "x", "dd")
vim.keymap.set("n", "y", "yy")
vim.keymap.set("n", "p", "pp")
```

引数の文字列だけ順に直したいとき:

```vim
f"
ci"delete<Esc>
n
.
```

または同種の式なら `ci"` と `.` を繰り返す。

### Markdown でよくある例

箇条書きをチェックボックスに変える:

```md
- task1
- task2
- task3
```

```vim
^r[ ]<Esc>
j
.
j
.
```

単語を backtick で囲む:

```vim
ysiw`
w
.
```

### 向いているケース

- 同じ変更を複数箇所へ適用したい
- 検索で次の対象へ飛べる
- text object で対象を安定して取れる

### 向いていないケース

- 毎回入力内容が大きく違う
- 複数ステップを丸ごと再現したい
- 位置取りまで含めて自動化したい

そういうときはマクロの方が向いている。

## Macro

dot-repeat で届かない「複数ステップ＋位置取り」をまとめて自動化する。
記録（record）→ 再生（playback）が基本。

### 基本操作（この設定）

`q` の誤記録を防ぐため、`q` 単押しでは記録が始まらない。

| キー    | 動作                                          |
| ------- | --------------------------------------------- |
| `qq`    | q レジスタへ記録開始（記録中の `q` で停止）    |
| `Q`     | q レジスタを再生（`@q` 相当）                 |
| `3Q`    | 3 回再生（`3@q` 相当。回数を前置できる）      |
| `99Q`   | エラーが出るまで再生（実質「最後まで」）      |
| `@@`    | 直前に実行したマクロをもう一度                |
| `"qp`   | q レジスタの中身を貼り付け（記録内容の確認）  |
| `"qyy`  | 行を q レジスタへ入れ直す（マクロの手直し）   |


### 記録のコツ

うまく繰り返せるマクロは「1 行分の処理 + 次行への移動」で組む。
上級者が口を揃えて言う要点は「文字数・桁数に依存しない動きで組む」こと。

- **行頭から始める** — 先頭で `0` か `^` を押し、開始位置を固定する。行のどこから再生しても同じ結果になる
- **`x` や `hl` の連打を避ける** — 行ごとに文字数が違うと崩れる。`dw`・`f{char}`・`/pattern` のように「意味のある単位」で動く
- **最後に `j`（必要なら `j0`）** — 再生するたび次の行へ進む
- **検索を混ぜると自走する** — マクロ内に `/pattern<CR>` を入れると、再生のたびに次の対象へ自分で飛ぶ（飛び石の対象に強い）
- **ヤンクは名前付きレジスタへ** — マクロ内で `y` する場合は `"ay` のように名前付きに退避する。無名レジスタ任せだとマクロを連鎖したとき内容が上書きされる
- **回数は多めに `99Q`** — マクロは「検索失敗」「行末超え」などのエラーで自動停止するので、多めに指定して止まるに任せる。回数が読めないときは下の再帰マクロが上位互換

### 再帰マクロ（回数を指定しない）

上級者の定番。マクロの末尾に自分自身（`@q`）を呼ばせ、エラーで止まるまで自走させる。
`99Q` のように回数を当てずっぽうで指定しなくてよい。

```vim
qqq             " q レジスタを空にする（前の記録の残骸を消す）
qq              " 記録開始
I"<Esc>A",<Esc> " 1 行分の処理
j               " 次の行へ
@q              " 自分自身を呼ぶ（記録中は空なので一度は不発でよい）
q               " 記録停止
Q               " 一度実行すれば最後まで自走する
```

`j` が最終行で失敗 → `@q` の連鎖が止まる、という仕組み。
「対象が何行あるか分からない」「検索で飛び石に当てる」ケースで特に効く。

### dot-repeat との使い分け

| 観点               | dot-repeat (`.` / `cgn`) | マクロ (`qq` … `Q`)        |
| ------------------ | ------------------------ | -------------------------- |
| 1 ステップの変更   | ◎ 最速                   | ○ 大げさ                   |
| 複数ステップの変更 | △ 1 操作にまとまらない   | ◎ 丸ごと記録できる         |
| 位置取りの自動化   | × `n` で手動移動         | ◎ `/検索` を記録に含められる |
| 回数指定           | × 手で連打               | ◎ `99Q` で一気に流す       |
| 範囲・全マッチ一括 | △                        | ◎ `:normal @q` / `:g`      |

### よくあるユースケース

#### 複数の変更を 1 行にまとめる

文字列リストを配列要素にする（接頭辞 `"` ＋ 接尾辞 `",` の 2 箇所を編集）。

```ts
apple
banana
cherry
```

```vim
qq             " 記録開始
I"<Esc>        " 行頭に "
A",<Esc>       " 行末に ",
j              " 次の行へ
q              " 記録停止
99Q            " 残りの行に一気に適用
```

結果:

```ts
"apple",
"banana",
"cherry",
```

接頭辞と接尾辞の 2 箇所を同時に直すため、`.` 1 回では再現できない典型例。

#### 検索を混ぜて飛び石の対象を処理する

離れた位置にある `log` の行だけコメントアウトする。

```ts
log(a)
ok()
log(b)
ok()
log(c)
```

```vim
qq              " 記録開始
/log<CR>        " 次の log へジャンプ
I// <Esc>       " 行頭にコメント
q               " 記録停止
99Q             " 一致が尽きるまで（検索失敗で停止）
```

マクロが自分で次の一致へ飛ぶので、`n` を手で押す必要がない。
`.` でも近いことはできるが、間に `n` を挟む手間が残る。

> この設定の `/` は very magic（`/\v`）。`(` `+` `|` などはメタ文字扱いになるため、
> リテラルとして探すならマクロ内でも `\(` のようにエスケープする。

#### 行を複製して連番を振る

`.` は同じ値しか入れられないが、マクロに `<C-a>`（数値インクリメント）を含めると 1 ずつ増える。

```
test_case_1
```

```vim
qq          " 記録開始
yyp         " 行を複製（カーソルは複製行の先頭へ）
<C-a>       " 行内の次の数値をインクリメント
q           " 記録停止
4Q          " さらに 4 行生成
```

結果:

```
test_case_1
test_case_2
test_case_3
test_case_4
test_case_5
```

`<C-a>` はカーソル位置から次の数値を探して増やすため、桁が変わっても破綻しない。

### 範囲・全マッチへ一括適用

記録済みの `@q` は Ex コマンドからも呼べる。行を巡る `j` が不要になる。

```vim
:'<,'>normal @q       " ビジュアル選択した行すべてに適用
:g/TODO/normal @q     " TODO を含む行だけに適用
:%normal @q           " ファイル全行に適用
```

### マクロの手直し

記録をやり直さず、中身を編集して使い回せる。

```vim
"qp        " q レジスタの内容を行に貼り出す
           " （行を直接編集）
"qyy       " 直した行を q レジスタへ入れ直す
```

`<C-v><Esc>` で記録された `<Esc>`（`^[` と表示される）も貼り出されるので、消さずに編集する。

### 向いているケース

- 1 操作にまとまらない複数ステップの変更
- 検索やジャンプを含めて位置取りごと自動化したい
- 同じ手順を多数の行・マッチへ流したい

### 向いていないケース

- 1 ステップで終わる変更（`.` / `cgn` の方が速い）
- 毎回入力内容が大きく違う（記録が再利用できない）
- 単純な置換で済む（`:s/old/new/g` や `cgn` の方が速くて見通しがよい）

### この設定での選び方

道具が揃っているぶん、まず安い手から試すと速い。

1. **単純な文字列置換** → `:s` か `cgn` + `.`（`very magic` の `/` で対象を絞れる）
2. **同じ 1 操作を散らばった箇所へ** → `*` / `/` で飛んで `.`
3. **複数ステップ・位置取りが必要** → ここで初めてマクロ。回数が読めなければ再帰マクロ
4. **マッチ行だけに一括** → `:g/pat/normal @q`

マクロは強力だが記録のコストがある。「`.` で届くか」を先に考えると無駄打ちが減る。

## Search

この設定では検索キーマップを少し変えている。

| キー | 実際の挙動 |
| --- | --- |
| `/` | `very magic` 前方検索 |
| `?` | リテラル後方検索 (`very nomagic`) |
| `*` / `#` | カーソル下の単語で前方/後方検索 |
| `n` / `N` | 次/前の一致へ（`zzzv` 付きで画面中央に保つ） |
| `<Esc>` | 検索ハイライトをクリア |
| `<leader>nh` | 検索ハイライトをクリア |

つまり:

- `/` は正規表現を短く書きやすい
- `?` は上方向へ、記号をそのまま探したいときに向く
- 単語単位なら打ち込むより `*` が速い

### 検索で意識すること

- 検索は「探す」より「移動する」ために使う
- 完全一致を狙いすぎず、近くまで飛べれば十分
- 検索後は `n`, `N`, `f`, `w`, `b` で微調整する
- 毎回打ち直すより、1回検索して `n` / `N` を使い回す
- ジャンプ後に戻るために `<C-o>` をセットで使う

### 使い分け

#### `/` を使うとき

- 単語の一部で飛びたい
- 正規表現を使いたい
- 複数候補を順に見たい

例:

```vim
/func
/const\s+
/TODO|FIXME
```

`very magic` なので、`+`, `|`, `()` などを比較的そのまま書ける。

#### `?` を使うとき

- 上方向へ探したい
- 記号をそのまま探したい
- 正規表現として解釈されたくない

例:

```vim
?foo.bar
?items[0]
?user?.name
```

`?` は `very nomagic` にしているので、`.` や `[` をそのまま扱いやすい。

### 実践パターン

#### 移動

```vim
/handle
n
n
N
```

1回検索して、次候補は `n` / `N` で辿る。

#### 編集と組み合わせる

```vim
/old
cgnnew<Esc>
.
.
```

検索一致を順に同じやり方で書き換える。

#### ジャンプして戻る

```vim
/someFunction
gd
<C-o>
```

検索や定義ジャンプの後は `<C-o>` で戻る。

### 検索語の作り方

- 長い文字列を入れすぎない
- 2〜6文字くらいで候補が絞れれば十分
- 一意でなくても `n` で進めばいい
- 近距離なら `f{char}` や `t{char}` の方が速い

### 向いている流れ

1. `/` または `?` で近くへ飛ぶ
2. `n` / `N` で目的位置まで寄せる
3. `ciw`, `cgn`, `.` などの変更へつなげる

## Substitute（`:s`）

「同じ変更を、機械的に、範囲ごとまとめて」適用する一括置換。`cgn` が
「目視で 1 個ずつ」なのに対し、`:s` は「条件に合う箇所を一発で」処理する。

### 基本構文

```vim
:[range]s/pattern/replacement/[flags]
```

| 部分          | 意味                               |
| ------------- | ---------------------------------- |
| `range`       | 適用範囲（省略時は**現在行のみ**） |
| `pattern`     | 検索パターン（正規表現）           |
| `replacement` | 置換後の文字列                     |
| `flags`       | 動作オプション（`g` `c` `i` など） |

区切りの `/` は任意の文字に変えられる。パスのように `/` を含む置換では
`:s#old/path#new/path#` のように `#` を使うとエスケープが減る。

### range の指定

| 書き方        | 範囲                             |
| ------------- | -------------------------------- |
| （省略）      | 現在行のみ                       |
| `%`           | ファイル全体                     |
| `'<,'>`       | ビジュアル選択（`V` 選択後 `:`） |
| `1,10`        | 1〜10 行目                       |
| `.,+5`        | 現在行から下に 5 行              |
| `.,$`         | 現在行から末尾まで               |
| `/foo/,/bar/` | `foo` の行から `bar` の行まで    |

実戦では `%`（全体）と `'<,'>`（選択範囲）の 2 つをまず覚える。

### flags

| flag | 動作                                                   |
| ---- | ------------------------------------------------------ |
| `g`  | 行内の**すべて**の一致を置換（無いと各行 1 個目だけ）  |
| `c`  | 置換ごとに確認（`y`/`n`/`a`/`q`/`l`）                  |
| `i`  | 大文字小文字を無視                                     |
| `I`  | 大文字小文字を区別                                     |
| `n`  | 置換せず**一致数を数える**だけ（件数の事前確認に便利） |

> この設定では `gdefault` を有効にしていないため、行内全置換には `g` が要る。
> `:%s/old/new/g` の末尾 `g` を付け忘れると各行 1 個目しか変わらない。

### この設定での注意（重要）

`/` キーは `\v`（very magic）に、`?` キーは `\V`（very nomagic）にリマップして
いるが、**これはキーマップであって `:s` には効かない**。`:s` のパターンは
Vim 標準の magic で解釈される。

そのため `:s` で very magic を使いたいなら、パターン先頭に `\v` を**明示**する。

```vim
:%s/\v(\w+)_(\w+)/\2_\1/g   " very magic を明示（() + が素直に使える）
:%s/\(\w\+\)_\(\w\+\)/\2_\1/g   " 明示しないと () + は \( \+ とエスケープが必要
```

迷ったら **`\v` を付ける** と `/` 検索と同じ感覚で書ける。

> Neovim 標準で `inccommand` が有効なので、入力中にハイライトと結果が
> ライブプレビューされる。打ちながら一致を確認できる。

### replacement の小技

| 記法       | 意味                                       | 例                                |
| ---------- | ------------------------------------------ | --------------------------------- |
| `\1`〜`\9` | キャプチャグループ（`\v` なら `(...)`）    | `:%s/\v(\w+)_(\w+)/\2_\1/`         |
| `&` / `\0` | 一致した文字列全体                         | `:%s/error/[&]/g` → `[error]`     |
| `\U` / `\L` | 以降を大文字 / 小文字化（`\E` で解除）     | `:%s/\v<(\w)/\U\1/g`（語頭大文字） |
| `\u` / `\l` | 次の 1 文字だけ大文字 / 小文字化           |                                   |
| `\=expr`   | 置換を式評価で生成                         | `:%s/\d\+/\=submatch(0)+1/`        |
| `\r`       | 改行を挿入                                 | `:%s/;/;\r/g`（`;` ごとに改行）    |
| `~`        | 直前の `:s` で使った replacement を再利用  |                                   |

`\zs` / `\ze`（一致の開始・終了位置をずらす）も `:s` で有効。
前の「箇条書き → チェックボックス」はこれが効く:

```vim
:%s/\v^(\s*)- /\1- [ ] /     " インデントを保ったまま "- " だけ置換
:%s/^\s*\zs- /- [ ] /        " \zs 版（\zs 以降が置換対象）
```

### 実践パターン

#### 件数を数えてから置換する

```vim
:%s/old//gn        " まず一致数だけ確認（置換しない）
:%s/old/new/g      " 問題なければ実行
```

#### 確認しながら置換する

```vim
:%s/old/new/gc     " 各箇所で y/n。cgn より速く全体を回せる
```

#### 直前の検索語をそのまま置換する

`/` で検索した後、パターンを空にすると**直前の検索語**が使われる。

```vim
/funcName<CR>      " まず検索（very magic）
:%s//replaced/g    " pattern 省略 → funcName が対象
```

#### 選択範囲だけに適用する

```vim
V}                 " 段落を選択
:s/foo/bar/g       " :'<,'> が自動で入る → 選択行だけ置換
```

#### 引数の文字列を一括で変える

前の Lua の例。最後の引数だけ狙う:

```vim
:%s/\v, "[^"]*"\)/, "delete")/   " 末尾の "..." を "delete" に
```

### `:g` と組み合わせる

`:g/pat/cmd` は「`pat` に一致する行だけで `cmd` を実行」。`:s` と相性がよい。

```vim
:g/TODO/s/^/DONE: /     " TODO を含む行だけ先頭に DONE: を付ける
:g/^$/d                 " 空行を全削除
:v/keep/d               " keep を含まない行を削除（:v は :g!）
```

### cgn / マクロ との使い分け

| 状況                       | 最適                          |
| -------------------------- | ----------------------------- |
| 規則的・件数多い・確認不要 | `:s`（一発、見通しがよい）    |
| 目視で 1 個ずつ確認したい  | `cgn` + `.`、または `:s///gc` |
| 各箇所で**違う**値に変える | `ci"` + 移動、またはマクロ    |
| 複数ステップ＋位置取り     | マクロ                        |
| 条件に合う**行だけ**に処理 | `:g/pat/...`                  |

ひとことで:

- **`:s`** = 同じ変更を機械的に一括
- **`cgn`** = 同じ変更を目視で飛び飛びに
- **マクロ** = 違う変更・複数ステップ

「規則性があるか」「確認が要るか」で `:s` か `cgn` かを選ぶ。
規則的なら `:s` が最短、と覚えておけばだいたい外さない。

## この設定固有のキーマップ

Vim デフォルトから変えているもの。定義は `keymaps.lua` と各プラグイン設定。

### 削除・貼り付け

| キー | 動作 |
| --- | --- |
| `x` / `X` | レジスタを汚さない削除オペレータ（`xx` で行、`xiw` で単語） |
| Visual `x` | 選択範囲をレジスタを汚さず削除 |
| Visual `p` | レジスタを保持して貼り付け |
| `y` / `p` / `P` | yanky のヤンクリングに記録される |
| `<C-p>` / `<C-n>` | 貼り付け直後にヤンク履歴を前後へ差し替え |

### 編集

| キー | 動作 |
| --- | --- |
| Visual `<` / `>` | インデント後に選択を維持 |
| `J` | 行結合してもカーソル位置を維持 |
| 空行で `i` / `A` | 自動インデント付きで Insert 開始 |
| `<leader>+` / `<leader>-` | 数値のインクリメント / デクリメント |

### テキストオブジェクト

| キー | 動作 |
| --- | --- |
| `i<space>` | `iW`（空白区切りの WORD） |
| `a"` `a'` `` a` `` | クォート込みで選択するが、後ろの空白は含まない（`2i"` 相当） |

### 移動

| キー | 動作 |
| --- | --- |
| `M` | 対応する括弧へジャンプ（`%` / `g%` を自動選択。デフォルトの「画面中央行へ移動」は無効化） |
| `<C-d>` / `<C-u>` | スクロール後カーソルを画面中央に保つ |

### マクロ（q プレフィックス）

`q` の誤記録を防ぐため、`q` 単押しでは何も起きない。

| キー | 動作 |
| --- | --- |
| `qq` | q レジスタへマクロ記録開始（記録中は `q` で停止） |
| `Q`  | q レジスタのマクロを実行 |
| `qo` | ウィンドウを 1 つにする (`:only!`) |
| `qt` | 直前のバッファへ切り替え (`<C-^>`) |

### Insert モード

| キー | 動作 |
| --- | --- |
| `<C-g><C-u>` | カーソル下の単語を大文字化 |
| `<C-g><C-l>` | カーソル下の単語を小文字化 |
| `<C-g><C-k>` | 単語の先頭文字を大文字化 |
| `<C-g><C-t>` | 単語の大小文字をトグル |

### パスのコピー

| キー | 動作 |
| --- | --- |
| `<leader>cp` | 相対パスをコピー |
| `<leader>cP` | 絶対パスをコピー |
| `<leader>cn` | ファイル名をコピー |
| `<leader>cN` | 拡張子なしのファイル名をコピー |
