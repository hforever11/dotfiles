# Lazygit チートシート

ターミナル UI の Git クライアント [lazygit](https://github.com/jesseduffield/lazygit) のよく使う操作まとめ。

設定ファイル: `config/lazygit/config.yml`

## 起動

| 方法              | コマンド / キー        |
| ----------------- | ---------------------- |
| ターミナルから    | `lazygit`              |
| Neovim から開く   | `<leader>lg`           |
| 現在ファイルのログ| `<leader>lf`           |

> Neovim 連携: lazygit 内で `e`（編集）すると、起動元の Neovim で該当ファイル・行が開く。
> delta のハイパーリンク（`lazygit-edit://`）も Neovim にジャンプする。

## Files（ファイル）パネル

| キー       | 説明                                   |
| ---------- | -------------------------------------- |
| `<space>`  | ステージ / アンステージ（ファイル単位）|
| `a`        | 全ファイルをステージ / 解除            |
| `c`        | コミット                               |
| `A`        | 直前のコミットに追加（amend）          |
| `C`        | git commit のエディタで詳細コミット    |
| `d`        | 変更を破棄（discard）                  |
| `e`        | エディタで開く                         |
| `s`        | 全変更を stash                         |
| `S`        | stash オプション                       |
| `o`        | OS デフォルトアプリで開く              |
| `<enter>`  | ファイルを開いて行単位ステージ         |

## ステージング（行・hunk 単位）

`<enter>` でファイルに入った状態。

| キー       | 説明                          |
| ---------- | ----------------------------- |
| `<space>`  | 行 / hunk をステージ・解除    |
| `v`        | 範囲選択モード                |
| `<left>`/`<right>` | hunk 間を移動         |
| `d`        | 選択行を破棄                  |
| `e`        | hunk をエディタで編集         |
| `<tab>`    | ステージ済み / 未ステージ切替 |

## Branches（ブランチ）パネル

| キー       | 説明                              |
| ---------- | --------------------------------- |
| `<space>`  | チェックアウト                    |
| `n`        | 新規ブランチ作成                  |
| `d`        | ブランチ削除                      |
| `r`        | 選択ブランチへ rebase             |
| `M`        | 選択ブランチを merge              |
| `f`        | fast-forward                      |
| `o`        | プルリクエスト作成                |
| `<enter>`  | コミット一覧を表示                |

## マージとコンフリクト解決

Branches パネルで `M`（merge）/ `r`（rebase）するとコンフリクトすることがある。
コンフリクトしたファイルは **Files パネルに赤字（`UU` 等）** で並ぶ。

### マージビューに入る

| キー       | 説明                                       |
| ---------- | ------------------------------------------ |
| `<enter>`  | コンフリクトファイルを開きマージビューへ   |

> up/down が効かないときは、まだ Files 一覧にいる（メインビューに入れていない）。
> 対象ファイルで `<enter>` を押すとメインビューにフォーカスが移る。

### マージビュー内（塊ごとに ours/theirs を選ぶ）

| キー                | 説明                                         |
| ------------------- | -------------------------------------------- |
| `<up>` / `<down>`   | 塊内の選択肢（ours / theirs）を移動          |
| `<left>` / `<right>`| 前 / 次のコンフリクト塊へ移動                |
| `<space>`           | ハイライト中の側を採用して解決               |
| `b`                 | 両方採用                                     |
| `e`                 | nvim で手動編集                              |
| `z`                 | 直前の解決を undo                            |
| `<esc>`             | マージビューを抜けて Files へ戻る            |

### 全ファイル解決後

| 状況         | 操作                                  |
| ------------ | ------------------------------------- |
| マージ中     | Files で `c`（マージコミット作成）    |
| リベース中   | `m`（Rebase メニュー）→ `continue`    |
| 中断したい   | `m` → `abort`                         |

## Commits（コミット）パネル

| キー       | 説明                                  |
| ---------- | ------------------------------------- |
| `r`        | コミットメッセージを編集（reword）    |
| `s`        | 下のコミットと squash                 |
| `f`        | 下のコミットへ fixup                  |
| `d`        | コミット削除（drop）                  |
| `e`        | この地点から interactive rebase       |
| `p`        | pick                                  |
| `A`        | ステージ変更を amend で取り込む       |
| `C`        | コミットをコピー（cherry-pick 用）    |
| `V`        | コピーしたコミットを貼り付け（cherry-pick）|
| `<ctrl+j>` / `<ctrl+k>` | コミットを上下に移動     |
| `g`        | reset オプション                      |
| `<space>` | revert                                 |
| `W`        | diff 比較メニュー（2 つの ref を比較）|
| `<enter>`  | コミット内のファイル一覧を表示        |

## Stash パネル

| キー       | 説明                          |
| ---------- | ----------------------------- |
| `<space>`  | stash を適用（apply）         |
| `g`        | pop（適用して削除）           |
| `d`        | stash を削除                  |
| `n`        | stash から新規ブランチ作成    |

## Push / Pull / Fetch

| キー       | 説明                          |
| ---------- | ----------------------------- |
| `p`        | pull                          |
| `P`        | push                          |
| `f`        | fetch                         |

> `git.disableForcePushing: true` のため force push は無効化されている。

## よく使う操作フロー

- **コミットする**: Files で `a`（全ステージ）→ `c`（メッセージ入力）→ 確定
- **直前コミットを修正**: 変更を `a` → Commits で対象に合わせ `A`（amend）
- **コミットメッセージ修正**: Commits で対象を選び `r`
- **コミットをまとめる**: Commits で `s`（squash）/ `f`（fixup）
- **やり直し（interactive rebase）**: Commits で起点に `e` → 各コミットで `p`/`s`/`d` → 確定
- **一時退避**: Files で `s`（stash）→ 戻すときは Stash で `g`（pop）
- **変更を捨てる**: Files で `d`（discard）
